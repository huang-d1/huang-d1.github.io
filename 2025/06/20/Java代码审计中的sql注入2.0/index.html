预编译下的sql注入

原理

预编译是将sql语句参数化，可预编译的语句，如 where语句中的内容是被参数化的。这就是说，预编译仅仅只能防御住可参数化位置的sql注入。那么，对于不可参数化的位置，预编译将没有任何办法。

不可参数化的位置：

  1. 表名、列名
  2. order by、group by
  3. limit
  4. join
  5. 等

如何防御

所以，对于sql注入存在两种情况，可参数化的，不可参数化的。

对于可参数化没商量，直接预编译解决一切。

而对于不可参数化的，只能通过设置白名单，过滤特殊符号，通过加引号强制转为字符串等方式进行拦截。
